# **Jupyter + Spark on Apple Silicon (Docker)**

This guide shows how to run Apache Spark with JupyterLab on Apple Silicon Macs using Docker. The `jupyter/all-spark-notebook` image provides a complete data science environment with Spark 3.x, Python libraries (pandas, NumPy, scikit-learn), and support for Scala and R—all running natively on ARM64 architecture.

This setup is ideal for local development, learning Spark, or prototyping data pipelines before deploying to production clusters.

## Technical Specifications

| Aspect | Details |
|--------|---------|
| **Image** | `jupyter/all-spark-notebook:latest` |
| **Platform** | ARM64 (Apple Silicon M1/M2/M3) |
| **Deployment** | Single-node local environment |
| **Components** | Apache Spark 3.x, PySpark, Scala, R, JupyterLab, Hadoop client libraries, Python scientific stack (pandas, NumPy, matplotlib, scikit-learn) |
| **Capabilities** | Interactive data analysis, distributed computing (local mode), notebook development, Spark SQL, DataFrame operations |

### Component Details

| Component | Details |
|-----------|---------|
| **Spark** | Apache Spark 3.x (local[*] mode) |
| **Languages** | Python (PySpark), Scala, R |
| **Interface** | JupyterLab |
| **Libraries** | NumPy, Pandas, Matplotlib, SciPy, scikit-learn, statsmodels |

### What You Can Do

- Run Spark DataFrames and Spark SQL queries locally
- Execute PySpark transformations and actions
- Develop and test notebooks in Python, Scala, or R
- Prototype data pipelines before production deployment
- Connect to external Spark clusters if needed

## Setup

### Prerequisites

- Docker Desktop running on macOS

### 1. Network Setup (one-time)

Create a custom Docker network to enable container-to-container communication:

```bash
docker network create dasnet
```

### 2. Pull Image

Download the official Jupyter Spark image optimized for ARM64:

```bash
docker pull jupyter/all-spark-notebook:latest
```

### 3. Run Container

Start the container with port mapping for JupyterLab:

```bash
docker run -it \
  --name jupyter-spark \
  --network dasnet \
  -p 8888:8888 \
  jupyter/all-spark-notebook:latest
```

**Access:** The terminal will display a URL with an access token (e.g., `http://127.0.0.1:8888/?token=xxxx`). Open this link in your browser to access JupyterLab.

### 4. Verify Spark

Create a new Python notebook in JupyterLab and test the Spark installation:

```python
from pyspark.sql import SparkSession
spark = SparkSession.builder.getOrCreate()
spark
```

If you see a SparkSession object with details about the Spark version and configuration, everything is working correctly.

## VS Code Integration

For those who prefer VS Code over the browser interface:

1. Install the **Dev Containers** extension
2. Open Command Palette: `Dev Containers: Attach to Running Container`
3. Select `jupyter-spark`

This connects VS Code directly to the running container, allowing you to create and edit notebooks with full IDE features.

## Container Management

Basic commands for managing the container lifecycle:

```bash
# Stop
docker stop jupyter-spark

# Remove
docker rm jupyter-spark

# Restart
docker start jupyter-spark
```

## Automation Script

To streamline the setup process, create a shell script that handles Docker checks, network creation, and container startup automatically.

Save this as `run_spark.sh`:

```bash
#!/bin/bash
set -e

docker info >/dev/null 2>&1 || { echo "Start Docker Desktop first"; exit 1; }

docker network create dasnet 2>/dev/null || true
docker rm -f jupyter-spark 2>/dev/null || true
docker pull jupyter/all-spark-notebook:latest

docker run -it \
  --name jupyter-spark \
  --network dasnet \
  -p 8888:8888 \
  jupyter/all-spark-notebook:latest
```

```bash
chmod +x run_spark.sh && ./run_spark.sh
```

Make it executable and run:

```bash
chmod +x run_spark.sh
./run_spark.sh
```

The script will handle all setup steps automatically, including checking if Docker is running, creating the network, and starting the container.
